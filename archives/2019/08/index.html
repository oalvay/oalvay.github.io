回首

吴恩达机器学习课的大名在很早以前就知道了，也早在一年前就注册了这门课程。不过因为我那时比较迷茫，在这门课和CMU大名鼎鼎的15-213之间抉择了很久后，选择了后者，因为当时我觉得全面了解一下计算机系统的基础原理比较好，而机器学习相比之下算是比较细的方向。

于是，我花了去年一整个暑假在15-213上面。学习过程其实很痛苦，编程作业连看答案都要花老半天才看得懂，写就更不可能写的出来了，而学习成果我感觉至少短期内会是屠龙之技。现在我边打字边反思了一下，可能这就是初学者学这门课应该有的反应，而这门课也不是学一遍就能懂的课；如果此时我再去学一遍的话，学习的收获一定会比学第一遍多非常多。

不过都学这么辛苦了，自然不可能完全是浪费时间。回想了一下去年底在kaggle上学的6门课和刚刚在coursera学完的这门，在学习速度和吸收效率上都提升了很多，可能就是拜15-213所赐。

小结

这门课难度偏易，不过学起来也不至于无聊到想快进，我基本都是一倍速看的。吴恩达上课讲得很细心，概念讲得很到位；编程作业也是尽心尽力，框架都给你做好了，剩个核心让你填。为人诟病的数学部分其实也还好，他讲到数学部分的时候跳过的部分都会提醒一下，这样有兴趣的就可以自己去课外了解。

这门课对我而言比较蛋疼的地方就是太偏理论，同时因为是门老课的原因，内容跟不上时代。举几个例：

  • 没有用python
  • ↑所以也没有教实用的算法包（比如Scikit-learn）
  • 没有讲树形算法
  • 没有讲NLP
  • 没有讲图模型

还有其他等等……

总的来说还是很受用的，我会推荐给想要学习机器学习，但又没有编程基础的人。

岔路口

下一步要做啥？其实我心里已经有了几个比较具体的选择：

  • 学特征工程，然后上kaggle打比赛
  • 上fast.ai学深度学习
  • 学Coursera的概率图模型

想了想还是选择了第一个，因为我觉得先把表格数据玩透比较好，同时学特征工程的经验可以用在非结构化数据上，再加上在kaggle打比赛也比较容易出成果。

那么问题来了，去哪里学特征工程？我曾经想直接去读kaggle上的冠军答案来学特征工程，但到真正开始试着读的时候，却发现不知道从何读起。结果还是觉得去找网课来系统地学习比较好，以下是我找到的看起来比较满意的两个：

  • Feature Engineering for Machine Learning
  • How to Win a Data Science Competition: Learn from Top Kagglers

以及囊括了特征工程相关资源的网站，作者是上面第一个网课的老师。比较好玩的一点是她居然把自己的课排在了推荐网课里的第一位，有点王婆卖瓜的感觉。

还有一点就是因为这门是付费课程，本来比较担心博客笔记会不会侵权的问题，于是就稍微上网查了一下，同时看了Udemy官网的政策；结论是应该没问题，我也会在接下来每个博客后标注出处。

以上就是我在过渡期的思考了，接下来就又是一段漫长的旅程，希望在结束时能在kaggle上有个能看的排名。
